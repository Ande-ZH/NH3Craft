# 08 玩家交易、市场、摊位与拍卖

> 返回：[总目录](index.md)

## 交易方式总览

服务器鼓励玩家自由交易，但通过税、限额、价格走廊和审计防止洗钱、刷成交和垄断。

| 交易方式 | 说明 |
| --- | --- |
| `/pay <玩家> <金额>` | 玩家间直接转账 |
| QuickShop-Hikari | 玩家商店/摊位 |
| AuctionHouse | 拍卖行，出售稀有物品 |
| ShopGUI+ 中央银行 | 系统价柜台，价格锚点 |

## `/pay` 转账

### 准入条件

- 达到 Tier 1 及以上。
- 有效在线 ≥ 72 小时。

### 转账规则

| 规则 | 数值 |
| --- | --- |
| 转账税 | 3%（被销毁，不进入任何玩家账户） |
| 最低转账额 | 10 XP |
| 每日对同一收款人上限 | 100,000 XP |
| 同一对玩家转账冷却 | 60 秒 |
| 同 IP 转账 | 直接禁止，有审计日志 |

### 每 Tier 转账上限

| Tier | 单笔最大转账额 |
| --- | --- |
| T1 | 50,000 XP |
| T2 | 100,000 XP |
| T3 | 300,000 XP |
| T4 | 1,000,000 XP |
| T5 | 无上限 |

> 转账税 3% 直接从转账金额中扣除并销毁。例如转账 1000 XP，对方收到 970 XP，30 XP 被销毁。

## 玩家商店（QuickShop-Hikari）

### 摊位数量限制

| Tier | 可开设商店数 |
| --- | --- |
| T0 | 0 |
| T1 | 3 |
| T2 | 10 |
| T3 | 30 |
| T4 | 60 |
| T5 | 100 |

### 商店交易税

- 每笔交易收取 **5% 交易税**，税入服务器回收。

## 拍卖行（AuctionHouse）

| 规则 | 数值 |
| --- | --- |
| 上架费 | 成交价的 2% |
| 成交税 | 成交价的 3% |
| 上架数量限制 | 按 Tier 限制（具体见游戏内） |

## 中央银行价格走廊

ShopGUI+ 中央银行同时提供系统收购价和出售价，作为市场价格锚点：

```text
系统收购价 = 价格下限（地板价）
系统出售价 = 价格上限（天花板价）
玩家自由交易 = 在上下限之间波动
```

作用：
- 防止基础材料价格归零。
- 防止关键材料被炒到新玩家买不起。
- 给市场一个稳定的价格锚点。

## 玩家建议

- **不要用极端价格给朋友转钱**——会被市场审计标记。
- 交易高价值物品时保留聊天或交易记录截图。
- 商人要计算 5% 交易税 + 3% 成交税在内，确保还有利润。
- 同 IP 内不能互相 `/pay`，家人/多开玩家请用不同 IP 或走拍卖行。
- 稳定供货比短期炒价更安全——中央银行的价格走廊会限制极端价格。
